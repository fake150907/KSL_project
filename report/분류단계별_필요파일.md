# 분류단계별 필요 파일

## 목적

AIHub `WORD morpheme` 라벨 탐색부터 최종 subset manifest 생성까지, 각 단계에서 어떤 파일이 왜 필요한지 정리한 문서임.

핵심 흐름은 아래와 같음.

```text
전체 후보 분석
-> 생활형 후보 검토
-> 최종 라벨 선택
-> 선택 라벨의 실제 sample_id 목록 생성
-> keypoint subset 결과 정리
```

## 전체 절차 한눈에 보기

| 순서 | 단계 | 목적 | 핵심 파일 |
|---|---|---|---|
| 1 | 전체 라벨 분석 | morpheme 전체에서 어떤 라벨이 있는지 확인 | `label_candidates.csv` |
| 2 | 생활형 후보 검토 | 데모에 쓸 만한 생활형 표현만 추려 보기 | `selected_lifestyle_candidates.csv` |
| 3 | 최종 라벨 선택 | 이번 MVP에서 실제 사용할 소수 라벨 확정 | `selected_labels_small.json` |
| 4 | target sample 생성 | 선택된 라벨에 해당하는 실제 sample_id 목록 생성 | `selected_label_targets.csv` |
| 5 | subset 결과 정리 | 실제 추출된 keypoint subset과 메타데이터 정리 | `sample_subset_manifest.csv` |
| 6 | 결과 형식 기준 | manifest에 들어갈 컬럼 형식 기준 제공 | `sample_subset_manifest_template.csv` |

## 현재 로컬 기준 데이터 개수

| 파일 또는 소스 | 현재 개수 | 설명 |
|---|---:|---|
| `data/raw` 아래 morpheme JSON 원본 | `9079` | 현재 로컬에 내려받아 둔 morpheme 샘플 수 |
| `label_candidates.csv` | `1723` | 고유 라벨 수 |
| `selected_lifestyle_candidates.csv` | `52` | 생활형 후보로 추린 라벨 수 |
| `selected_labels_small.json` | `8` | 최종 선택 라벨 수 |
| `selected_label_targets.csv` | `75` | 선택 라벨에 대응하는 target sample 수 |
| `sample_subset_manifest.csv` | `60` | 실제 subset manifest 행 수 |

참고:

- AIHub에서 제공한 경량화 `new_sample` 파일은 본 프로젝트 핵심 흐름에서 사용하지 않음
- 이유는 생활형 MVP 핵심 표현과 성격이 다른 라벨이 다수 포함되어 있었기 때문임
- 실제 예시로 `골키퍼`, `권투`, `낚시`, `운전면허정지`, `견제하다`, `성토`, `울보`, `그립다` 같은 항목이 확인되었음

## 단계별 설명

### 1. 전체 라벨 분석

사용 파일:

- [label_candidates.csv](C:/github/ai-project-01/data/label_candidates.csv)

의미:

- AIHub morpheme JSON 전체를 훑어서 발견한 라벨 후보 목록임
- 어떤 단어/표현이 존재하는지 먼저 넓게 보는 단계임
- 소스 데이터는 현재 로컬 기준 `data/raw` 아래 morpheme JSON `9079개`임
- 결과 파일 행 수는 현재 `1723개`임

주요 컬럼:

| 컬럼 | 의미 |
|---|---|
| `label` | 발견된 라벨 이름 |
| `sample_count` | 해당 라벨이 몇 번 등장했는지 |
| `example_ids` | 예시 sample id 일부 |

샘플:

```text
label,sample_count,example_ids
걷다,30,NIA_SL_WORD1295_REAL01_D_morpheme|NIA_SL_WORD1295_REAL01_F_morpheme|NIA_SL_WORD1295_REAL01_L_morpheme
지도,20,NIA_SL_WORD0236_REAL01_D_morpheme|NIA_SL_WORD0236_REAL01_F_morpheme|NIA_SL_WORD0236_REAL01_L_morpheme
결점,20,NIA_SL_WORD0922_REAL01_D_morpheme|NIA_SL_WORD0922_REAL01_F_morpheme|NIA_SL_WORD0922_REAL01_L_morpheme
가다,20,NIA_SL_WORD0943_REAL01_D_morpheme|NIA_SL_WORD0943_REAL01_F_morpheme|NIA_SL_WORD0943_REAL01_L_morpheme
```

이 단계에서 보는 질문:

- 어떤 라벨이 존재하는가
- 자주 나오는 라벨은 무엇인가
- 생활형 데모로 쓸 만한 표현이 있는가

### 2. 생활형 후보 검토

사용 파일:

- [selected_lifestyle_candidates.csv](C:/github/ai-project-01/data/selected_lifestyle_candidates.csv)

의미:

- 전체 후보 중에서 생활형 데모에 적합하다고 판단한 라벨만 추려본 중간 검토 파일임
- 아직 최종 확정 전 단계임
- 사람이 보기 쉽게 압축한 후보군임
- 소스는 `label_candidates.csv` 임
- 현재 파일 행 수는 `52개`임

이 단계에서 보는 질문:

- 이 프로젝트 분석 대상 선정으로 적당한가?
  - 즉,  일상 표현으로 보여주기 좋은가?
- 의미가 직관적인가?


예시 후보:

```text
가다
감사
괜찮다
배고프다
병원
아프다
우유
자다
```

### 3. 최종 라벨 선택

사용 파일:

- [selected_labels_small.json](C:/github/ai-project-01/data/selected_labels_small.json)

의미:

- 이번 MVP에서 실제로 사용할 최종 라벨 목록임
- 코드와 후속 추출 작업이 이 파일을 기준으로 돌아감
- 소스는 `selected_lifestyle_candidates.csv` 검토 결과임
- 현재 선택된 라벨 수는 `8개`임

샘플:

```json
{
  "labels": [
    "감사",
    "괜찮다",
    "아프다",
    "병원",
    "배고프다",
    "우유",
    "자다",
    "가다"
  ],
  "mode": "quick_test_real_morpheme",
  "min_samples": 5,
  "note": "Selected from real AI Hub WORD morpheme labels. Keypoint data still required for real training."
}
```

이 단계에서 보는 질문:

- 이번 실험에서 정확히 어떤 라벨을 쓸 것인가
- 너무 많지 않게, 그러나 시연 가능한 범위로 정했는가

### 4. target sample 생성

사용 파일:

- [selected_label_targets.csv](C:/github/ai-project-01/data/selected_label_targets.csv)

의미:

- `selected_labels_small.json` 에 들어 있는 최종 라벨들에 대해 실제로 해당하는 모든 `sample_id` 를 펼쳐놓은 파일임
- 작업자가 keypoint zip 안에서 찾아야 할 실무용 기준표임
- 소스는 `selected_labels_small.json` + morpheme 원본임
- 현재 target sample 수는 `75개`임

샘플:

```text
sample_id,label,angle,morpheme_path,start,end,duration,split
NIA_SL_WORD0943_REAL01_D,가다,D,data/raw/.../NIA_SL_WORD0943_REAL01_D_morpheme.json,1.217,2.224,2.7,train
NIA_SL_WORD0943_REAL01_F,가다,F,data/raw/.../NIA_SL_WORD0943_REAL01_F_morpheme.json,1.217,2.224,2.7,train
NIA_SL_WORD0943_REAL01_L,가다,L,data/raw/.../NIA_SL_WORD0943_REAL01_L_morpheme.json,1.217,2.224,2.7,train
NIA_SL_WORD0943_REAL01_R,가다,R,data/raw/.../NIA_SL_WORD0943_REAL01_R_morpheme.json,1.217,2.224,2.7,train
NIA_SL_WORD0943_REAL01_U,가다,U,data/raw/.../NIA_SL_WORD0943_REAL01_U_morpheme.json,1.217,2.224,2.7,validation
```

이 단계에서 보는 질문:

- 최종 선택한 라벨에 해당하는 실제 샘플은 무엇인가?
- 작업자가 어떤 sample id를 찾아야 하는가?
- 각 샘플의 촬영 각도와 발화 구간은 무엇인가?

중요:

- 작업자에게 전달할 기준표는 `label_candidates.csv` 가 아니라 이 파일임

### 5. subset 결과 정리

사용 파일:

- [sample_subset_manifest.csv](C:/github/ai-project-01/data/sample_subset_manifest.csv)

의미:

- 실제로 keypoint subset 추출이 끝난 뒤 결과를 정리한 manifest임
- 전처리와 학습은 이 파일을 기준으로 진행함
- 소스는 `selected_label_targets.csv` 를 기준으로 추출한 keypoint subset임
- 현재 manifest 행 수는 `60개`임

대표 컬럼:

| 컬럼 | 의미 |
|---|---|
| `sample_id` | 샘플 고유 ID |
| `label` | 정답 라벨 |
| `angle` | 촬영 각도 |
| `morpheme_path` | 연결된 morpheme JSON 경로 |
| `start` | 발화 시작 시각 |
| `end` | 발화 종료 시각 |
| `duration` | 전체 영상 길이 |
| `split` | train / validation |
| `keypoint_path` | 추출된 keypoint 폴더 경로 |
| `is_dummy` | 더미 데이터 여부 |

샘플 구조:

```text
sample_id,label,angle,morpheme_path,start,end,duration,split,keypoint_path,is_dummy
NIA_SL_WORD0943_REAL01_D,가다,D,...,1.217,2.224,2.7,train,...,False
```

이 단계에서 보는 질문:

- 실제 subset이 잘 추출되었는가
- 전처리와 학습에 넘길 준비가 되었는가

### 6. manifest 형식 기준

사용 파일:

- [sample_subset_manifest_template.csv](C:/github/ai-project-01/data/sample_subset_manifest_template.csv)

의미:

- `sample_subset_manifest.csv` 를 만들 때 따라야 하는 컬럼 형식 기준임
- 작업자들이 결과를 맞춰서 전달하도록 돕는 템플릿임
- 템플릿 파일이라 데이터 행 수 자체가 핵심은 아님

## 파일 분류 요약

| 분류 | 파일 | 소스 | 현재 개수 | 역할 |
|---|---|---|---:|---|
| 분석용 | `label_candidates.csv` | morpheme JSON 원본 `9079개` | `1723` | 전체 라벨 후보 탐색 |
| 검토용 | `selected_lifestyle_candidates.csv` | `label_candidates.csv` | `52` | 생활형 후보 압축 |
| **선택용** | `selected_labels_small.json` | `selected_lifestyle_candidates.csv` 검토 결과 | `8` | 최종 라벨 확정 |
| <mark>작업자용</mark> | `selected_label_targets.csv` | `selected_labels_small.json` + morpheme 원본 <br><mark>수어 class를 늘리고 시키고 싶으면, 선택용파일을 먼저 변경하고, 그 파일기준으로 작업자용파일을 다시 생성함</mark> | `75` | 실제 추출할 sample_id 목록 |
| 결과용 | `sample_subset_manifest.csv` | `selected_label_targets.csv` 기반 subset 추출 | `60` | 추출 완료된 subset 결과 |
| 기준용 | `sample_subset_manifest_template.csv` | 결과 CSV 설계 기준 | - | 결과 CSV 형식 기준 |

## 최종 흐름 요약

```text
label_candidates.csv
-> selected_lifestyle_candidates.csv
-> selected_labels_small.json
-> selected_label_targets.csv
-> sample_subset_manifest.csv
```

## 한 줄 정리

- `label_candidates.csv` 는 전체 후보를 보는 파일임
- `selected_lifestyle_candidates.csv` 는 생활형 후보를 검토하는 파일임
- `selected_labels_small.json` 는 이번 MVP의 최종 선택 라벨 파일임
- `selected_label_targets.csv` 는 작업자가 실제로 찾아야 하는 sample id 목록임
- `sample_subset_manifest.csv` 는 실제 추출 완료 결과 파일임
