# 1안 MediaPipe 재학습 vs 2안 OpenPose 호환 재매핑 비교

작성일: 2026-04-27

## 1. 현재 결론

이번 흐름은 아래처럼 정리된다.

```text
2안: MediaPipe -> OpenPose 호환 재매핑
     기존 OpenPose checkpoint 그대로 사용
     빠른 확인용 실험

1안: AIHub 원본 mp4 -> MediaPipe landmark 재추출
     MediaPipe 데이터로 CNN-GRU 재학습
     정석 파이프라인
```

2안은 이미 `report/2안_연구결과_해석.md`에서 종료 판단이 내려진 실험이다.  
우리가 이번에 이어서 수행한 작업은 1안, 즉 MediaPipe 기준 재학습이다.

## 2. 2안 결과 요약

2안은 MediaPipe landmark를 기존 OpenPose 기반 모델이 기대하는 BODY_25 + 양손 21개 구조에 맞춰 넣는 방식이었다.

평가 대상은 `sample_video`의 영문 클립 16개였다.

| 지표 | 결과 |
|---|---:|
| final accuracy | 25.00% |
| average accuracy | 31.25% |
| majority accuracy | 31.25% |

2안에서 반복적으로 어려웠던 라벨:

```text
괜찮다
병원
아프다
```

해석:

- 좌표 순서 문제를 줄이는 데는 의미가 있었다.
- 하지만 OpenPose로 학습한 모델과 MediaPipe 입력 사이의 분포 차이는 여전히 남았다.
- 그래서 2안만으로는 최종 해결책이 되기 어렵다.

## 3. 1안 결과 요약

1안은 AIHub 원본 mp4를 다시 받아 MediaPipe Holistic landmark로 전처리하고, 그 데이터로 CNN-GRU 모델을 재학습한 방식이다.

사용 데이터:

| 항목 | 값 |
|---|---:|
| 라벨 수 | 8 |
| 전체 샘플 수 | 75 |
| train | 60 |
| validation | 15 |
| sequence length | 32 |
| feature count | 201 |

학습 결과:

| 지표 | 결과 |
|---|---:|
| best epoch | 42 |
| validation accuracy | 73.33% |

이 결과만 보면 2안보다 1안이 더 타당하다. 학습 데이터와 추론 입력을 모두 MediaPipe 기준으로 맞췄기 때문이다.

## 5. 왜 이런 차이가 났나

1안의 validation accuracy는 AIHub 원본 영상에서 나눈 validation set 기준이다.  
반면 `label8_clip`은 별도 촬영 또는 편집 클립에 가까워 보이고, AIHub 원본 validation과 촬영 조건, 프레이밍, 동작 구간, 사람, 배경이 다를 수 있다.

즉 현재 상황은 이렇게 보는 것이 맞다.

```text
AIHub 내부 validation:
MediaPipe 재학습 모델이 어느 정도 학습됨

외부/편집 클립:
도메인 차이 때문에 일반화가 아직 약함
```

또한 학습 샘플 수가 라벨별로 균형적이지 않다.

| 라벨 | 샘플 수 |
|---|---:|
| 가다 | 20 |
| 감사 | 5 |
| 괜찮다 | 5 |
| 배고프다 | 10 |
| 병원 | 5 |
| 아프다 | 5 |
| 우유 | 10 |
| 자다 | 15 |

`가다` 샘플이 가장 많고, validation에서도 `가다` recall이 높았다. 외부 클립에서 `가다` 쏠림이 나타난 것은 이 불균형과 도메인 차이가 함께 작용했을 가능성이 크다.

## 6. 비교표

| 항목 | 2안 OpenPose 호환 재매핑 | 1안 MediaPipe 재학습 |
|---|---|---|
| 원본 mp4 필요 | 필요 없음 | 필요 |
| 기존 checkpoint 사용 | 사용 | 새로 학습 |
| 좌표계 | MediaPipe를 OpenPose처럼 재배열 | 학습/추론 모두 MediaPipe |
| 장점 | 빠르게 실험 가능 | 구조적으로 더 올바른 방향 |
| 한계 | 분포 차이 해결 부족 | 데이터 수와 일반화가 아직 부족 |
| 내부/클립 평가 | 16클립 average/majority 31.25% | AIHub validation 73.33%, label8_clip 0.00% |
| 현재 판단 | 종료 | 계속 개선 |

주의할 점은 두 평가 세트가 완전히 같지 않다는 것이다. 따라서 위 표는 절대적인 성능 순위라기보다, 각 방식의 현재 관찰 결과로 봐야 한다.

## 7. 다음 작업

우선순위는 아래 순서가 좋다.

1. `label8_clip` 또는 직접 촬영 웹캠 클립을 학습 데이터 쪽에 추가한다.
2. 8개 라벨 샘플 수를 균형 있게 맞춘다.
3. 특히 `감사`, `괜찮다`, `병원`, `아프다`는 최소 15~20개 이상 확보한다.
4. 학습 시 class weight 또는 balanced sampler를 적용해 `가다` 쏠림을 줄인다.
5. 같은 `label8_clip`으로 재평가해서 0%에서 개선되는지 확인한다.
6. 웹캠 테스트에서는 top-1만 보지 말고 top-3와 window 안정성도 함께 기록한다.

## 8. 최종 판단

2안은 실패라기보다 방향 전환의 근거를 만든 실험이다.  
1안은 구조적으로 맞는 방향이지만, 현재 데이터만으로는 외부 클립 일반화가 부족하다.

따라서 지금부터의 핵심은 모델 구조 변경보다 데이터 보강이다.

```text
현재 최우선:
MediaPipe 기준 샘플을 더 모으고,
라벨별 균형을 맞춘 뒤,
같은 평가 클립으로 재학습 전후를 비교한다.
```
