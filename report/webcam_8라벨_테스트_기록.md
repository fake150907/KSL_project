# 웹캠 8라벨 테스트 기록

작성일: 2026-04-27

## 목적

MediaPipe 기준으로 재학습한 8라벨 sequence 모델이 실제 웹캠 입력에서 얼마나 안정적으로 동작하는지 확인한다.

이번 테스트에서 가장 먼저 볼 것은 정확도보다 `window=32/32`가 안정적으로 채워지는지다. 손 검출이 끊기면 sequence 모델이 예측할 기회를 얻지 못한다.

## 테스트 전 확인

서버:

```text
Flask: http://127.0.0.1:5000
React/Vite: http://127.0.0.1:3001
```

브라우저에서 접속:

```text
http://127.0.0.1:3001
```

권장 설정:

| 항목 | 권장값 |
|---|---|
| model | sequence |
| window size | 32 |
| confidence threshold | 0.35 |
| stable min count | 2 |
| max missing frames | 3 |

카메라 조건:

- 손과 상반신이 화면 중앙에 들어오게 한다.
- 손이 화면 밖으로 나가지 않게 한다.
- 배경과 손 색이 너무 비슷하지 않게 한다.
- 너무 가까이 붙지 말고 손 전체가 보이게 한다.
- 랜드마크 overlay에서 양손 또는 동작에 필요한 손이 계속 찍히는지 본다.

| `raw_prediction_rows` | 모델 raw 예측이 실제로 나온 row 수 |

## 테스트 방법

각 라벨마다 5회씩 수행한다.

1. 카메라 앞에서 준비 자세를 잡는다.
2. 손 landmark가 안정적으로 찍히는지 확인한다.
3. 한 라벨 동작을 5회 반복한다.
4. 각 회차마다 화면의 top-1/top-3 결과를 기록한다.
5. 테스트 후 로그 요약 명령을 실행한다.

## 판정 기준

| 상황 | 해석 | 다음 조치 |
|---|---|---|
| `window=32/32`가 거의 안 됨 | 모델 문제가 아니라 입력 안정성 문제 가능성이 큼 | 카메라 거리, 조명, 손 위치 조정 |
| top-3 안에 정답이 자주 있음 | 후처리/threshold/stabilization 조정 여지 있음 | threshold와 stable count 조정 |
| top-3에도 정답이 거의 없음 | 데이터 분포 또는 샘플 부족 문제 가능성 | 해당 라벨 샘플 추가 |
| 특정 라벨로 계속 쏠림 | 라벨 불균형 또는 동작 유사도 문제 가능성 | confusion 기록 후 데이터 보강 |

## 현재 관찰

현재 Flask 로그에서는 손 검출이 간헐적으로 들어오지만, `window=32/32`까지 안정적으로 도달한 예측이 부족하다.

따라서 다음 테스트에서는 먼저 아래 상태를 목표로 한다.

```text
has_hand=True가 연속으로 유지됨
window가 1/32 -> 32/32까지 증가함
top-3 후보가 화면과 로그에 표시됨
```

## 현재 로그 자동 요약

`scripts/summarize_webcam_predictions.py`로 현재 Flask 로그를 분석한 결과는 다음과 같다.

| 항목 | 값 |
|---|---:|
| prediction log rows | 2570 |
| hand detected rows | 2041 |
| hand detected ratio | 79.4% |
| max window progress | 32/32 |
| window filled rows | 1708 |
| raw prediction rows | 1637 |
| stable label rows | 1637 |

top-1 후보 분포:

| top-1 | count |
|---|---:|
| 가다 | 1112 |
| 우유 | 327 |
| 감사 | 185 |
| 배고프다 | 13 |

해석:

- 전체 로그 기준으로는 `window=32/32`가 도달하고 예측도 생성되고 있다.
- 다만 top-1이 `가다`에 많이 쏠려 있다.
- 실제 테스트에서는 각 라벨을 따로 수행한 시간대를 구분해 기록해야 한다.
- 현재 자동 로그만으로는 사용자가 어느 라벨을 수행했는지 알 수 없으므로, `data/webcam_8label_test_trials.csv`에 기대 라벨과 화면 top-3를 함께 적어야 한다.
