# 수어 MVP 경량 데모 작업자 역할분담 가이드

## 1. 현재 목표

전체 AI Hub WORD 데이터를 모두 처리하지 않고, 먼저 발표 가능한 경량 데모를 만든다.

이번 단계의 목표는 다음 8개 생활형 클래스만 대상으로 keypoint subset을 모아 모델 성능과 웹캠 데모 가능성을 확인하는 것이다.

```text
가다
감사
괜찮다
배고프다
병원
아프다
우유
자다
```

## 2. 전체 작업 흐름

작업 순서는 아래와 같다.

```text
1. morpheme에서 8개 클래스의 sample id 목록을 기준으로 삼는다.
2. 각 작업자는 맡은 keypoint zip을 다운로드한다.
3. 각 keypoint zip 안에서 8개 클래스에 해당하는 sample만 추출한다.
4. 추출한 subset 폴더와 manifest 파일을 한 곳으로 모은다.
5. 통합 PC에서 subset을 병합한다.
6. 전처리, 학습, 평가를 다시 실행한다.
7. Streamlit UI와 웹캠 데모에 반영한다.
```

## 3. 작업자별 역할

### 공통 기준

중요한 현재 전제:

- 현재 로컬에서 확보한 morpheme은 `REAL01` 기준만 있다고 보고 작업한다
- 따라서 현재 [selected_label_targets.csv](C:/github/ai-project-01/data/selected_label_targets.csv) 도 `REAL01` sample id 기준으로만 생성되어 있다
- 이 상태에서는 `01_real_word_keypoint.zip` 은 바로 target 추출이 가능하지만, `02_real_word_keypoint.zip` 같은 `REAL02` zip은 현재 morpheme만으로는 직접 매칭되지 않는다

각 작업자는 클래스를 나눠 맡지 않는다.

대신 keypoint zip 번호를 나눠 맡는다. 이유는 특정 클래스의 샘플이 어느 keypoint zip에 들어 있는지 미리 보장할 수 없기 때문이다.

예를 들어 `우유` 클래스가 05번 zip에만 있다고 볼 수 없고, 02번, 08번, 13번 등 여러 zip에 나뉘어 있을 수 있다.

따라서 모든 작업자는 자신이 맡은 zip 안에서 동일한 8개 클래스를 찾는다.

## 4. 추천 분담 방식

현재 Training WORD keypoint zip은 총 16개이다.

현재 01번은 이미 처리했으므로, 남은 대상은 02번부터 16번까지 총 15개이다.

5명이 작업한다면 아래처럼 나눈다.

```text
작업자 A: 01, 02, 03, 04 keypoint zip
작업자 B: 05, 06, 07 keypoint zip
작업자 C: 08, 09, 10 keypoint zip
작업자 D: 11, 12, 13 keypoint zip
작업자 E: 14, 15, 16 keypoint zip
```

6명이 작업한다면 아래처럼 나눌 수 있다.

```text
작업자 A: 01, 02, 03 keypoint zip
작업자 B: 04, 05 keypoint zip
작업자 C: 06, 07, 08 keypoint zip
작업자 D: 09, 10 keypoint zip
작업자 E: 11, 12, 13 keypoint zip
작업자 F: 14, 15, 16 keypoint zip
```

## 5. 작업자가 실제로 해야 할 일

각 작업자는 아래 4가지만 하면 된다.

```text
1. 맡은 keypoint zip 다운로드
2. 압축 해제 또는 부분 추출 준비
3. 8개 클래스에 해당하는 sample id만 추출
4. 추출 결과를 통합 담당자에게 전달
```

작업자는 전체 keypoint 데이터를 모두 학습하지 않는다.

작업자의 산출물은 학습 모델이 아니라 subset 데이터이다.

작업 시작 전에 통합 담당자는 아래 명령으로 현재 8개 클래스의 전체 target sample 목록을 먼저 생성한다.

```powershell
python scripts/export_selected_targets.py
```

생성 파일은 아래와 같다.

```text
data/selected_label_targets.csv
```

현재 기준 target 수는 총 75개이다.

```text
가다      20
감사       5
괜찮다      5
배고프다    10
병원       5
아프다      5
우유      10
자다      15
```

이 CSV는 작업자들이 어떤 sample id를 찾아야 하는지 확인하는 기준표이다.

주의:

- 현재 이 기준표는 `REAL01` 대상표라고 이해해야 한다
- 즉 문서상 분담 방식과 실제 로컬 데이터 상태는 다를 수 있다
- `02~16` zip을 바로 같은 방식으로 처리하려면, 해당 `REALxx`에 대응하는 morpheme 또는 별도 매칭 기준이 먼저 필요하다

## 6. 작업자가 전달해야 하는 결과물

각 작업자가 최종 전달해야 하는 것은 아래 두 가지이다.

```text
1. selected_keypoints 폴더
2. sample_subset_manifest.csv 파일
```

예상 폴더 구조는 다음과 같다.

```text
worker_A_result/
├─ selected_keypoints/
│  ├─ NIA_SL_WORDxxxx_REALxx_F/
│  │  ├─ NIA_SL_WORDxxxx_REALxx_F_000000000000_keypoints.json
│  │  ├─ NIA_SL_WORDxxxx_REALxx_F_000000000001_keypoints.json
│  │  └─ ...
│  └─ NIA_SL_WORDyyyy_REALxx_U/
│     └─ ...
└─ sample_subset_manifest.csv
```

`sample_subset_manifest.csv`에는 최소한 아래 정보가 들어가야 한다.

```text
sample_id
label
angle
start
end
duration
keypoint_path
```

공통 컬럼 템플릿은 아래 파일을 기준으로 한다.

```text
data/sample_subset_manifest_template.csv
```

현재 프로젝트에서 사용하는 전체 공통 컬럼은 다음과 같다.

```text
sample_id,label,angle,morpheme_path,start,end,duration,split,keypoint_path,is_dummy
```

각 컬럼의 의미는 다음과 같다.

```text
sample_id      : 영상, morpheme, keypoint를 연결하는 공통 샘플 ID
label          : 정답 클래스 이름
angle          : 촬영 각도, F/U/D/R/L 중 하나
morpheme_path  : 해당 morpheme JSON 경로
start          : 수어 표현 시작 시각
end            : 수어 표현 종료 시각
duration       : 전체 영상 길이
split          : train 또는 validation
keypoint_path  : 추출된 keypoint JSON 폴더 경로
is_dummy       : 실제 AI Hub 데이터면 False
```

작업자는 결과를 전달하기 전에 아래 검증 스크립트를 실행한다.

```powershell
python scripts/validate_subset_manifest.py --manifest worker_A_result/sample_subset_manifest.csv
```

현재 PC에서 `keypoint_path` 폴더가 실제로 존재하는지까지 확인하려면 아래처럼 실행한다.

```powershell
python scripts/validate_subset_manifest.py --manifest worker_A_result/sample_subset_manifest.csv --check_paths
```

검증 결과가 `[OK] Manifest validation passed.`로 나오면 통합 담당자에게 전달한다.

## 6-1. 작업자가 실제로 실행할 명령 예시

작업자 A가 `02_real_word_keypoint.zip`을 처리한다고 가정하면 아래처럼 실행한다.

```powershell
python -m src.data.extract_keypoint_subset_from_zip `
  --zip_path "data/raw/aihub_downloads/word_keypoint_02/004.수어영상/1.Training/라벨링데이터/REAL/WORD/02_real_word_keypoint.zip" `
  --output_dir "data/raw/worker_A_selected_keypoints_02" `
  --manifest_output "data/worker_A_sample_subset_manifest_02.csv"
```

같은 방식으로 03번, 04번 zip도 각각 따로 실행한다.

```powershell
python -m src.data.extract_keypoint_subset_from_zip `
  --zip_path "data/raw/aihub_downloads/word_keypoint_03/004.수어영상/1.Training/라벨링데이터/REAL/WORD/03_real_word_keypoint.zip" `
  --output_dir "data/raw/worker_A_selected_keypoints_03" `
  --manifest_output "data/worker_A_sample_subset_manifest_03.csv"
```

```powershell
python -m src.data.extract_keypoint_subset_from_zip `
  --zip_path "data/raw/aihub_downloads/word_keypoint_04/004.수어영상/1.Training/라벨링데이터/REAL/WORD/04_real_word_keypoint.zip" `
  --output_dir "data/raw/worker_A_selected_keypoints_04" `
  --manifest_output "data/worker_A_sample_subset_manifest_04.csv"
```

각 zip 처리 후에는 아래처럼 manifest 검증을 수행한다.

```powershell
python scripts/validate_subset_manifest.py --manifest data/worker_A_sample_subset_manifest_02.csv
python scripts/validate_subset_manifest.py --manifest data/worker_A_sample_subset_manifest_03.csv
python scripts/validate_subset_manifest.py --manifest data/worker_A_sample_subset_manifest_04.csv
```

작업자가 여러 zip 결과를 하나로 합치고 싶으면 아래 명령을 사용한다.

```powershell
python scripts/merge_subset_manifests.py `
  --inputs data/worker_A_sample_subset_manifest_02.csv data/worker_A_sample_subset_manifest_03.csv data/worker_A_sample_subset_manifest_04.csv `

```

통합 담당자는 모든 작업자 manifest를 받아 같은 방식으로 최종 병합하면 된다.

## 7. sample id의 의미

sample id는 영상, morpheme, keypoint를 서로 연결하는 공통 이름이다.

예시는 다음과 같다.

```text
NIA_SL_WORD0943_REAL01_F
```

각 부분의 의미는 다음과 같다.

```text
NIA_SL       : AI Hub 수어 데이터 prefix
WORD0943    : WORD 데이터의 단어/표현 번호
REAL01      : 실제 촬영 데이터 제공자 또는 촬영 구분 번호
F           : 촬영 각도
```

촬영 각도는 다음과 같다.

```text
F: 정면
U: 위쪽
D: 아래쪽
R: 오른쪽
L: 왼쪽
```

같은 sample id를 기준으로 morpheme과 keypoint가 연결된다.

```text
morpheme:
NIA_SL_WORD0943_REAL01_F_morpheme.json

keypoint:
NIA_SL_WORD0943_REAL01_F_000000000000_keypoints.json
NIA_SL_WORD0943_REAL01_F_000000000001_keypoints.json
...
```

keypoint 파일명의 숫자 부분은 프레임 번호이다.

```text
000000000000: 첫 번째 프레임
000000000001: 두 번째 프레임
000000000002: 세 번째 프레임
```

## 8. 통합 담당자가 할 일

통합 담당자는 작업자들이 전달한 subset을 한 PC에 모은다.

그 다음 아래 작업을 수행한다.

```text
1. selected_keypoints 폴더 병합
2. sample_subset_manifest.csv 병합
3. 중복 sample_id 제거
4. keypoint_path 경로 정리
5. 전처리 실행
6. baseline 모델 학습
7. GRU/LSTM 모델 학습
8. 평가 결과 확인
9. Streamlit UI에 최신 checkpoint 반영
10. 웹캠 데모 테스트
```

1번부터 5번까지는 모델 종류와 무관한 공통 데이터 준비 작업이다.

즉, 이후에 RandomForest, MLP, LSTM, GRU, Transformer 중 어떤 분석 모델을 사용하더라도 `selected_keypoints` 병합, manifest 병합, 중복 제거, 경로 정리, 전처리까지는 동일하게 필요하다.

6번부터는 모델별 학습 단계이다. baseline은 빠르게 성능 가능성을 확인하기 위한 모델이고, GRU/LSTM은 시퀀스 동작을 더 잘 반영하기 위한 MVP 최종 후보 모델이다.

## 9. 현재 01번 처리 결과

현재 01_real_word_keypoint.zip은 이미 처리되었다.

현재 확보한 경량 subset은 다음 기준으로 구성되어 있다.

```text
대상 클래스: 8개
총 샘플 수: 60개
시퀀스 길이: 32 프레임
입력 feature: pose + left hand + right hand 2D keypoint
face keypoint: MVP에서는 제외
```

현재 baseline 모델은 start/end 구간을 반영한 keypoint 전처리 기준으로 약 0.8 수준의 validation 정확도를 확인했다.

단, 샘플 수가 아직 작기 때문에 이 정확도는 최종 성능이 아니라 가능성 확인용 수치로 봐야 한다.

## 10. 원천 영상의 역할

원천 데이터는 실제 수어 영상 mp4이다.

현재 학습은 원천 mp4를 직접 사용하지 않고, keypoint JSON 좌표를 사용한다.

원천 영상은 아래 용도로 사용한다.

```text
1. 사람이 어떤 동작을 했는지 눈으로 확인
2. 휴대폰에서 영상을 재생한 뒤 PC 웹캠으로 비춰 실시간 인식 테스트
3. 발표용 데모 장면 구성
```

즉, 학습에는 keypoint JSON이 핵심이고, 원천 영상은 데모 검증과 시각 확인에 유용하다.

## 11. 작업 완료 기준

각 작업자의 완료 기준은 다음과 같다.

```text
1. 맡은 keypoint zip 다운로드 완료
2. 8개 클래스에 해당하는 sample만 추출 완료
3. selected_keypoints 폴더 생성 완료
4. sample_subset_manifest.csv 생성 완료
5. 통합 담당자에게 압축 파일로 전달 완료
```

통합 작업의 1차 완료 기준은 다음과 같다.

```text
1. 전체 subset 병합 성공
2. 전처리 성공
3. baseline 모델 재학습 성공
4. validation 결과 출력 성공
5. Streamlit UI에서 예측 결과 출력 성공
6. 웹캠에서 MediaPipe landmark 추출 성공
7. 휴대폰으로 재생한 원천 영상에 대해 예측 클래스 출력 성공
```

## 12. 주의사항

전체 keypoint zip을 모두 학습 데이터로 쓰려고 하지 않는다.

이번 단계에서는 8개 클래스에 해당하는 sample만 추출한다.

각 작업자는 자신이 맡은 zip 안에서 8개 클래스를 모두 찾는다.

파일명 끝의 `F/U/D/R/L`은 JSON 안에 있는 값이 아니라 파일명에 들어 있는 촬영 각도 정보이다.

morpheme의 `start`, `end`는 전체 영상 길이가 아니라 해당 수어 표현이 등장하는 구간이다.

keypoint 프레임은 이 `start`, `end` 구간을 반영해서 모델 입력으로 변환한다.
