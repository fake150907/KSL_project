## 3. 데이터 전처리 담당자 변경 내용

기존 8개 MVP 작업에서는 한국어 `label` 기준으로 `y`를 만들었습니다.

기존 방식:

```python
label = row["label"]
y = label_to_id[label]
```

3000개 전체 작업에서는 반드시 `word_id` 기준으로 만들어야 합니다.

변경 방식:

```python
word_id = row["word_id"]
y = label_map_word_id_3000[word_id]
```

제공된 V2 소스:

```text
team_handover_2026-04-29/WORD_3000_CLASS_UPDATE_20260506/DATA_PREPROCESS_WORKER/src/data/preprocess_mediapipe_videos.py
```

이 소스는 `label_map_word_id_3000.json`을 읽어서 `WORD0001 -> 0`, `WORD1381 -> 1380`, `WORD3000 -> 2999` 기준으로 NPZ의 `y`를 생성합니다.

## 4. 취합/통합 담당자 변경 내용

기존 취합 소스는 shard 또는 FILE 결과를 합치면서 `row["label"]` 기준으로 `y`를 다시 만들었습니다.

기존 방식:

```python
labels = sorted(set(all_y_labels))
label_to_id = {label: index for index, label in enumerate(labels)}
y = np.asarray([label_to_id[label] for label in all_y_labels])
```

3000개 전체 작업에서는 이 방식이 위험합니다. 같은 한국어 라벨이 여러 `WORD ID`에 중복으로 존재하기 때문입니다.

변경 방식:

```python
word_id = row["word_id"]
y = label_map_word_id_3000[word_id]
```

제공된 V2 소스:

```text
team_handover_2026-04-29/WORD_3000_CLASS_UPDATE_20260506/INTEGRATION_MASTER/scripts/merge_files.py
team_handover_2026-04-29/WORD_3000_CLASS_UPDATE_20260506/INTEGRATION_MASTER/scripts/merge_shards.py
```

두 소스 모두 `labels`를 현재 데이터에 등장한 라벨 목록으로 만들지 않고, `WORD0001~WORD3000` 전체 순서로 고정합니다.

|---|---|

| `X` shape | `(N, 32, 225)` |

| `y` 범위 | `0~2999` |
| `sample_id`와 `y` | `sample_id` 안의 `WORDxxxx`와 `label_map_word_id_3000.json` 기준 class id가 일치해야 함 |

## 6. 왜 WORD ID 기준인가

전체 3000개 데이터에서는 같은 한국어 라벨이 여러 `WORD ID`에 연결될 수 있습니다.

예를 들어 `가다`, `자다` 같은 라벨은 하나의 한국어 단어처럼 보이지만, AIHub 데이터 안에서는 여러 `WORDxxxx`로 나뉘어 있습니다.

따라서 한국어 `label` 기준으로 `y`를 만들면 서로 다른 `WORD ID`가 하나의 클래스로 합쳐질 수 있습니다. 이번 3000개 전체 학습에서는 이를 막기 위해 `label`이 아니라 `word_id`를 기준으로 클래스 번호를 고정합니다.

핵심 요약:

> 좌표 추출 방식은 그대로 유지한다. 바뀌는 것은 NPZ의 정답 `y`를 `label` 기준이 아니라 `WORD ID` 기준으로 만드는 것이다.

## 7. 모델 학습 담당자 변경 내용

전처리/취합이 끝난 NPZ에는 이미 `y`가 `WORD ID` 기준 `0~2999`로 들어갑니다. 따라서 모델 학습 담당자는 `y`를 다시 만들 필요는 없습니다.

다만 학습 코드가 8개 MVP 기준으로 되어 있다면 반드시 3000개 기준으로 바꿔야 합니다.

수정해야 할 대표 지점:

| 수정 지점 | 기존 8개 MVP | 3000개 전체 |
|---|---|---|
| 클래스 수 상수 | `NUM_CLASSES = 8` | `NUM_CLASSES = 3000` |
| config | `max_classes: 8` | `max_classes: 3000` |
| 모델 마지막 출력층 | `Linear(..., 8)` | `Linear(..., 3000)` |
| label map 해석 | 한국어 라벨 기준 | WORD ID 기준 |
| 예측 결과 해석 | `2 -> 괜찮다` | `1380 -> WORD1381 -> 괜찮다` |

예시:

```python
labels = np.load(npz_path, allow_pickle=True)["labels"]
num_classes = len(labels)  # 3000
```

또는 고정값을 쓰는 코드라면:

```python
NUM_CLASSES = 3000
```

모델 마지막 출력층도 같이 바꿔야 합니다.

```python
self.classifier = nn.Linear(hidden_size, 3000)
```

예측 결과 해석은 `idx_to_label_3000.json`을 사용합니다.

```python
pred_id = 1380
result = idx_to_label_3000[str(pred_id)]
```

결과 예시:

```json
{
  "word_id": "WORD1381",
  "label": "괜찮다"
}
```

모델 학습 담당자용 별도 안내:

```text
team_handover_2026-04-29/WORD_3000_CLASS_UPDATE_20260506/MODEL_TRAINER/README.md
```
